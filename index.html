// What it does

Everything you need for a compliant APK audit

Eight modular analysis engines work in sequence to cover static analysis, dependency auditing, secret detection, and AI-assisted risk reporting — all from a single APK upload.

🔐

MASVS-Mapped Permissions

25 dangerous permissions classified by risk level (HIGH/MEDIUM) and mapped to their exact MASVS v2.1.0 requirement and chapter.

📦

Software Bill of Materials

Identifies 36 known Java/Kotlin library fingerprints and native .so files, annotated with vendor, category, and production-risk flag.

🔍

DEX String Analysis

Scans bytecode for API keys, JWT tokens, AWS credentials, Firebase URLs using 9 regex patterns. Secrets anonymized — never stored in plaintext.

🤖

AI Executive Summary

Google Gemini Flash generates a plain-language risk report with executive summary, key findings, and recommendations — the only open-source tool with this feature.

📋

Rule-Based Compliance

All 12 MASVS verdicts are entirely rule-based and traceable to specific evidence sources. The LLM layer never influences compliance scoring.

🚀

Zero-Config Deployment

Runs locally via python app.py or one-command Docker. No persistent server, no cloud dependency.

// Analysis pipeline

8-step automated audit workflow

From APK upload to auditor-ready HTML/JSON report in under 10 seconds.

M1 · apk_hasher

Hash Computation

SHA-256 & MD5 in streaming mode. Primary reproducibility identifier per MASVS-RESILIENCE-1.

M2 · manifest_parser

Manifest Extraction

Parses binary AndroidManifest via pyaxmlparser: debuggable, allowBackup, SDK levels, cleartext flags.

M3 · permission_analyzer

Permission Classification

Risk score s = min(100, 15·nHIGH + 7·nMED). Calibrated on 20 labeled APKs.

M4 · sbom_generator

SBOM Generation

36 library fingerprints + native .so detection. Annotated with vendor, category, and risk flag.

M5 · strings_analyzer

DEX String Analysis

9 compiled regex patterns. 8 behavioral indicators (SSL pinning, obfuscation, root detection…)

M6 · masvs_mapper

MASVS Compliance

12 requirements across Chapters 4, 6, 7, 9, 10, 14. Rule-based only — fully traceable.

M7 · gemini_client

AI Summarization

Gemini Flash at temperature 0.2. JSON Schema validated. Local fallback if API unavailable.

M8 · report_generator

Report Export

UUID-addressed HTML (audit-ready) + JSON (programmatic). Reproducible across environments.
